LAP Protocol Demo

This demonstrates the Linked Attestation Protocol (LAP) v0.2 - a system for proving publisher-resource association through cryptographic attestations.

1.
Resource Presence
Verifies the Resource Attestation is accessible, demonstrating intent to distribute
2.
Resource Integrity
Compares SHA-256 hashes to ensure content matches what was attested
3.
Publisher Association
Confirms the publisher controls the namespace containing the resource
üí°
Key Benefit: When all checks pass, we have cryptographic proof that the content came from the claimed publisher. Publishers can also dissociate by removing attestations, leaving no durable evidence of prior association.

Who's Doing What

This Client (Port 8081): Fetches fragment from Publisher, sends to Verifier, displays results
Publisher (Port 8080): Serves LAP fragments and attestations
Verifier (Port 8082): Performs the 3 verification checks and returns results
